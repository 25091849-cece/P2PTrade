# Contributing to P2PTrade

## Branching and Pull Request Policy

- Do not commit directly to `main`.
- Create a feature branch using the format `feature/<name>`.
- Push your changes to your feature branch.
- Open a pull request targeting `main`.
- At least one approval is required before merge.
- Repository code owner review is required.
- Only the repository owner/maintainer merges pull requests into `main`.

## Notes

Repository visibility, branch protection rules, and merge permissions are enforced in GitHub repository settings.
